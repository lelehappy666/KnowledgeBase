# ALIGNMENT: 展品知识库 (Exhibit Knowledge Base)

## 1. 项目背景与目标
**当前状态**: 空项目，已获取核心需求文档《展品展项知识库体系.pdf》。
**目标**: 开发一个**全功能的展品展项知识库系统**，用于沉淀公司的展品设计、工程实现、科学原理及产品化经验。
**部署环境**: 公司内部局域网 (Intranet)，完全离线，素材存储在 NAS。

## 2. 核心需求分析 (Based on User Input & PDF)

### 2.1 知识库结构 (四大维度)
系统数据模型需支撑以下四个维度的信息录入与检索：
1.  **A 认知基础库**:
    *   **趋势洞察**: 记录新技术、行业风格、生命周期。
    *   **分类体系**: 交互方式（触控/体感...）、内容类型（科普/艺术...）、体验类型。
    *   **评价体系**: 多维度打分（体验性、易理解性、成本等 1-5分）。
2.  **B 内容知识库 (教育/策展)**:
    *   **科学原理**: 关联知识点、学段、课程标准。
    *   **学科矩阵**: 展项 × 年级 × 学科。
    *   **叙事主题**: 关联主题（如能源、生命）。
3.  **C 技术与工程知识库**:
    *   **工程风险**: 安全性、环境干扰、维护要求。
    *   **技术模块**: 硬件清单、BOM成本结构。
4.  **D 研发与产品化**:
    *   研发SOP、产品化状态、失败案例复盘。

### 2.2 功能需求
1.  **展品管理 (CMS)**:
    *   创建/编辑展品，覆盖上述复杂字段。
    *   支持多维度标签和分类。
2.  **多媒体管理 (NAS集成)**:
    *   支持上传 **图片**、**视频**、**3D模型**。
    *   文件物理存储在 NAS 挂载目录，数据库存储路径引用。
    *   **自动分类**: 基于文件类型（MIME）自动归类（图片/视频/模型）。
3.  **检索与展示**:
    *   多维度筛选（按学科、按交互方式、按成本区间）。
    *   详情页展示完整画像（原理、工程、评价）。
4.  **用户系统**:
    *   简单的管理员登录（用于后台数据录入）。
    *   访客浏览模式。

### 2.3 技术栈 (已确认)
*   **Frontend**: Next.js 14+ (App Router), Tailwind CSS, Shadcn/UI。
*   **Backend**: Next.js Server Actions (自带 API 能力)。
*   **Database**: **SQLite** (via Prisma ORM) - 适合离线、零配置、单文件数据库，完美契合局域网需求。
*   **File Storage**: 本地文件系统 API (直接读写 NAS 挂载盘/本地目录)。

## 3. 关键决策确认
*   **数据结构策略**: 鉴于 PDF 字段极其丰富，数据库设计将采用 **核心表 (Exhibits) + 关联表 (Categories, Tags) + JSON 扩展字段 (Metadata)** 的混合模式，以平衡开发速度和灵活性。对于评价体系、工程参数等结构化弱或变动大的字段，使用 JSON 存储。
*   **3D预览**: 前端将集成 `Three.js` 或 `React-Three-Fiber` 用于简单的 3D 模型预览 (.glb/.gltf)。

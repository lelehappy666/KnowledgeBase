# ALIGNMENT: 学科对照库开发

## 1. 原始需求
用户要求开发"展项内容知识库"中的"学科对照库"模块。
- **功能参考**: `e:\KnowledgeBase\学科对照库.xlsx`。
- **核心功能**:
    - 新建学科对照。
    - **快速导入**: 复制 JSON 快速创建。
    - **存储规则**: 扁平化存储在 `Doc/Content_Knowledge/Subject_Mapping/` 下，文件名为 `{Name}.json`。

## 2. 需求理解与边界确认

### 2.1 数据模型
基于 Excel 分析，学科对照库包含以下 6 大组信息：
1.  **基础信息**: 矩阵 ID, 名称, 编码, 目的.
2.  **关联维度**:
    - **展项**: 类型, 子类, 交互, 体验.
    - **学科**: 分类, 分支, 主题, 课标.
    - **学段**: 学段, 年级, 年龄, 认知水平.
3.  **知识匹配**: 知识点 ID/名称/编码, 深度, 类型, 匹配度评分, 依据, 优劣势, 互补展项.
4.  **教学整合**: 时机, 方式, 时长, 衔接, 难度分析(认知/操作), 适应性(性别/文化/特殊需求), 群体适配.
5.  **目标评估**: 目标(知识/能力/素养/情感), 评估(维度/方法/工具/标准), 产出(作品/展示).
6.  **实施支持**: 条件(空间/设备/人员/安全/维护), 资源(指南/手册/资料), 时间, 成本, 案例验证.

### 2.2 功能范围
- **CRUD**: 增删改查学科对照条目。
- **JSON 导入**: 支持标准 JSON 模板导入。
- **文件存储**: 数据以 `{Name}.json` 形式存储在 `Doc/Content_Knowledge/Subject_Mapping/` 目录下。

### 2.3 技术约束
- **技术栈**: Next.js 14, TypeScript, Tailwind CSS, Shadcn UI.
- **存储**: 本地文件系统 (Node.js `fs`).
- **复用**: 复用现有的 `content-system.ts` 逻辑（需扩展支持 Subject Mapping）。

## 3. 疑问澄清
- **关联数据**: Excel 中提到了关联 A2 (展项类型) 和 B1 (知识点) 模块。
    - *决策*: 在 JSON 中存储关联的 ID (UUID) 或名称，暂不强制做强外键约束，但在 UI 上可以考虑未来做成选择器。目前版本主要以文本/Tag 输入为主。

## 4. 结论
需求清晰，数据结构已解析。将按照"文件系统 CMS"模式进行开发，结构与"科学原理库"保持高度一致。
